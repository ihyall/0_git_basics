1)
	1.
		I. Рабочий каталог - это текущая директория, подключённая к Git, в которой хранятся файлы, содержащиеся в текущей ветке.
		II. Репозиторий - папка, подключённая к Git.
		III. Коммит - операция отправки изменений в рабочем каталоге в удалённый репозиторий.

2)
	1. Удалённый репозиторий - сервер, на котором хранятся все коммиты.
	2. Комманды git нужно вводить в консоли одного из дистрибутивов git.
	3. Ветки нужны для разделения рабочего процесса по уровням и удобного перемещения между ними.
	4. Конфликты слияния возникают, когда две или более объединённые ветви пытаются производить разные изменения файла.
	5. Разрешается конфликт слияния путём редактирования конфликтующего файла и удаления маркеров конфликта.
3)
	1. Облачный хостинг репозиториев нужен для совместной работы нескольких пользователей в одном репозитории.
	2. github.com обладает функцией создания веток, добавления чужих веток к себе, а также создания и сравнения коммитов.
	3. Командная работа может осуществляться засчёт функции github pull request, которая позволяет нескольким пользователям работать в своих репозиториях, а затем сделать слияние работы каждого в один один проект.
